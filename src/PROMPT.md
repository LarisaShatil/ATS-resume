# Fast prompt for AI:

Act as a senior recruiter and ATS optimization expert.

Create an ATS-friendly resume based on the following data.

Requirements:

1. STRUCTURE (strict):
- Header (Name, Title)
- Contact Information (Phone, Email, Location, LinkedIn, GitHub)
- Professional Summary (3–4 lines, keyword-rich)
- Skills (grouped, relevant to role)
- Work Experience (reverse chronological)
- Projects (if relevant)
- Education
- Optional: Certifications, Languages

2. ATS RULES:
- Use standard section headings (Experience, Education, Skills)
- Ensure high keyword match with the job description
- Avoid icons or special characters
- Output must be ATS-readable plain text

3. BULLET POINTS (CRITICAL):
- Each bullet must follow: Action verb + task + measurable impact
- Add metrics where possible (%, time saved, users, performance)
- Avoid generic phrases like "responsible for"
- Avoid repetition of verbs (use synonyms)

Example:
❌ Built UI components
✅ Built 15+ reusable UI components, improving development speed by 30%

4. CONTENT OPTIMIZATION:
- Emphasize achievements over responsibilities
- Use strong action verbs (developed, optimized, implemented, enhanced)
- Remove filler words
- Keep bullets concise (1–2 lines max)

5. LANGUAGE:
- Professional, concise, recruiter-friendly
- No spelling or grammar mistakes

6. OUTPUT:
- Clean, well-structured resume
- ATS-optimized wording

Input data:
[PASTE RESUME OR USER DATA HERE]

Target job description:
[PASTE JOB DESCRIPTION HERE]
